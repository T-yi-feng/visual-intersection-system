# 专利核心差异说明（面向审查员）

## 一句话总结

> **对比专利**：计算车辆之间的"几何距离"来衡量交织
> **本专利**：计算车辆的"方向场卷积"来衡量交织

---

## 核心差异对照表

| 对比维度 | 对比专利 | 本专利 |
|---------|---------|--------|
| **计算什么** | 车辆对的相对位移 | 单车的方向场影响力 |
| **怎么计算** | 几何公式（距离、角度） | 卷积运算（场 × 核） |
| **需要什么** | 拥堵中心点 | 无需中心点 |
| **输出什么** | 车辆对的交织程度 | 单车的归因分数 |
| **能否GPU加速** | 困难 | 原生支持 |

---

## 三种根本性差异

### 差异一：计算范式不同

```
对比专利：几何计算范式
─────────────────────────
  车辆A ●───────● 车辆B
        ↘       ↙
         ↘     ↙
          ↘   ↙
           ↘ ↙
            ● 拥堵中心

  计算：d_AB = ||A - B||（欧氏距离）
  思路：距离越近 → 交织越强


本专利：场卷积范式
─────────────────────────
  ┌─────────────────────┐
  │ 方向场 O_k(x,y)     │    卷积核 K_k      影响力场 R_k
  │  ↑  ↑  ↑  ↑        │    ╱             =  ↑  ↑  ↑  ↑
  │  ↑  ↑  ↑  ↑        │   ╱                 ↗  ↗  ↗  ↗
  │  ↑  ↑  ↑  ↑        │  ╱（各向异性）       →  →  →  →
  └─────────────────────┘

  计算：R_k = O_k ⊛ K_k（卷积）
  思路：方向场经卷积扩展 → 得到路径影响力 → 冲突场相乘
```

**关键区别**：几何计算是"点对点"，场卷积是"场对场"

---

### 差异二：是否依赖拥堵中心

```
对比专利：需要定义拥堵中心
─────────────────────────
  问题1：拥堵中心在哪？
  - 需要聚类算法确定？
  - 需要人工标注？
  - 不同时间中心会移动？

  问题2：中心定义不一致会导致结果不同


本专利：无需拥堵中心
─────────────────────────
  思路：自底向上

  每辆车 → 散布到网格 → 构建方向场 → 卷积 → 冲突场 → 单车归因

  不需要任何"中心点"或"参考点"
  每辆车的归因只取决于：
  - 自身的运动方向（R[k_i]）
  - 所在位置的冲突强度（ΣR[k']）
```

**关键区别**：本专利是"无中心"的自底向上方法

---

### 差异三：输出粒度不同

```
对比专利：输出"车辆对"关系
─────────────────────────
  输出示例：
  ┌────────────────────────────┐
  │ 车辆对 (A,B) → 交织程度 0.8│
  │ 车辆对 (A,C) → 交织程度 0.3│
  │ 车辆对 (B,C) → 交织程度 0.5│
  └────────────────────────────┘

  问题：
  - N辆车有 N(N-1)/2 对关系
  - 难以回答"哪辆车贡献最大"


本专利：输出"单车"归因
─────────────────────────
  输出示例：
  ┌────────────────────────────┐
  │ 车辆 #123 → 归因分数 0.85  │ ← 最大贡献者
  │ 车辆 #456 → 归因分数 0.62  │
  │ 车辆 #789 → 归因分数 0.21  │
  └────────────────────────────┘

  优势：
  - N辆车只有 N 个归因值
  - 直接回答"哪辆车贡献最大"
  - 可用于关键车辆识别
```

**关键区别**：本专利输出更清晰、更可操作

## 向审查员陈述要点

### 要点1：计算范式本质不同

> "对比专利使用几何距离计算，本专利使用场卷积计算。这是两种完全不同的数学范式，就像'计算两点距离'和'计算两个函数的卷积'是不同的数学操作。"

### 要点2：不依赖拥堵中心

> "对比专利需要预先定义'拥堵中心'，本专利通过自底向上的场计算，完全不需要任何中心点或参考点，更加通用。"

### 要点3：输出粒度不同

> "对比专利输出'车辆对'的交织关系，本专利输出'单车'的归因分数。这使得本专利能够直接识别'哪辆车贡献最大'，而不仅仅是'哪两辆车有交织'。"

### 要点4：计算效率不同

> "本专利的卷积操作可以利用GPU并行加速，计算复杂度为O(G²)+O(N)，而对比专利的两两配对计算复杂度为O(N²)，当车辆数量多时效率差异显著。"

---

## 总结

| 本质差异 | 对比专利 | 本专利 |
|---------|---------|--------|
| **数学基础** | 欧氏几何 | 场论与卷积 |
| **计算对象** | 点对点距离 | 场对场卷积 |
| **参考依赖** | 需要中心点 | 无中心依赖 |
| **输出形式** | 关系矩阵 | 归因向量 |
| **GPU友好** | 否 | 是 |

**结论**：两专利虽然都涉及"车辆交织程度计算"，但采用的技术路线、数学基础、计算方法和输出形式存在本质差异，属于不同的技术方案。
